# Constraints SQL

Reglas que garantizan la calidad de los datos en [[SQL_DDL]].

*   **PRIMARY KEY (PK)**: Identificador único. Ver [[Clave_Primaria]].
*   **FOREIGN KEY (FK)**: Enlace a otra tabla. Ver [[Clave_Foranea]].
*   **NOT NULL**: Obliga a tener un valor.
*   **UNIQUE**: Impide valores duplicados.
*   **CHECK**: Valida una condición lógica.
*   **DEFAULT**: Asigna un valor por defecto.
