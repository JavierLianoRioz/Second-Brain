# Pasos del Diseño de una BD

El diseño de una [[Concepto de Base de Datos (BD)|base de datos]] es un proceso estructurado que consta de varias fases:

1.  **Análisis de Requisitos:** Entender las necesidades de información de los usuarios.
2.  **Diseño Conceptual:** Se crea un modelo de alto nivel, independiente del SGBD. Se utiliza el [[Modelo Entidad-Relación (MER)]].
3.  **Diseño Lógico:** Se transforma el modelo conceptual a un modelo de datos específico, habitualmente el [[Modelo Relacional]].
4.  **Diseño Físico:** Se especifica la implementación física en un [[Sistemas de Gestión de Bases de Datos (SGBD)|SGBD]] concreto (índices, almacenamiento, etc.).

---
**Relacionado:** [[Modelo Entidad-Relación (MER)]], [[Modelo Relacional]]
